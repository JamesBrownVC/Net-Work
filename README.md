# Account Conquest Room — Connection Fabric

AI-native revenue intelligence platform. This repo currently implements the
first slice of Phase 1 (Layer 0/1, the Connection Fabric): the unified
schema, the store, and one connector (`mockcrm`). The full product spec is
in [`ACR_PRD.md`](./ACR_PRD.md).

The previous Next.js prototype has been archived at
[`archive/nextjs-old-start/`](./archive/nextjs-old-start/) and is no longer
active. All new backend work happens in Python.

## Scope of this slice

Implemented:
- `fabric/protocol.py` — the `Connector` protocol every source implements (`health`, `pull`, `normalize`).
- `fabric/schema.py` — pydantic entities: `Company`, `Person`, `Interaction`, `Deal`, `Signal`, `OrgEdge`, `Transcript`, `Reference`.
- `fabric/store.py` — SQLAlchemy models for the unified schema plus idempotent upsert helpers keyed on natural keys (re-running ingest never duplicates rows).
- `fabric/connectors/mockcrm.py` — the only connector so far. The fixture *is* the source; there is no live mode.
- `scripts/seed.py` — deterministic generator (`random.seed(42)`) for `fixtures/mockcrm/accounts.json`: 25 customer accounts with log-normal ARR (EUR 8k–400k), owned products, renewal dates, and deal stage.
- `scripts/ingest.py` — pull → normalize → upsert, currently over `mockcrm` only.

Not yet implemented (later work, per `ACR_PRD.md`): the other seven
connectors (Gmail, Calendar, Slack, Notion, FullEnrich, Sillage, Gradium),
MCP server wrappers, the CLI, `config/apis.yaml`, engines (warmth,
allocator, fortress), agents, the Slack surface, and the Theater UI.
Mock-first only, no live APIs, no invented endpoints.

## Setup

Requires Python 3.12+.

```bash
make setup   # pip install -e ".[dev]"
make seed    # generate fixtures/mockcrm/accounts.json
make ingest  # pull -> normalize -> upsert into acr.db
make test    # pytest
```

Without `make` (e.g. plain PowerShell), run the equivalent commands
directly:

```powershell
python -m pip install -e ".[dev]"
python scripts/seed.py
python scripts/ingest.py
python -m pytest
```

Copy `.env.example` to `.env` if you want to override `DATABASE_URL`;
nothing is required for `mockcrm` to run.

## Mock-first design

`MOCK_MODE=true` is the default and, right now, the only mode. `mockcrm`
has no live adapter: the JSON fixture generated by `scripts/seed.py` is the
system of record for this connector. Every derived/generated value is
seeded (`random.seed(42)`), so `make seed` produces byte-identical output
every run.

## Adding a new connector

1. Implement the `Connector` protocol in `fabric/connectors/<name>.py`: `name`, `health()`, `pull(since)`, `normalize(raw)`.
2. Return `fabric.schema` entities from `normalize()` — pydantic validation is your contract test.
3. If the entity type is new, add an `upsert_<entity>()` to `fabric/store.py` and register it in `_UPSERT_DISPATCH`.
4. Add fixtures under `fixtures/<name>/` and a contract test at `tests/test_contract_<name>.py` following the pattern in `tests/test_contract_mockcrm.py`.
5. Append an instance to `CONNECTORS` in `scripts/ingest.py`.

## Tests

```bash
make test
```

- `tests/test_store.py` — upsert idempotency across `CompanyRow` and `DealRow`, and the generic `upsert_entity` dispatcher.
- `tests/test_contract_mockcrm.py` — connector health, deterministic pull, and schema-valid normalize output against the seeded fixture.
